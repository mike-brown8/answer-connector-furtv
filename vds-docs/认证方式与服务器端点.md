
# 认证方式与服务器端点

本页用于说明拿到签名后，业务端点如何携带令牌发起请求。  
接口文档未单独说明时，默认按本页认证方式执行。

## 1. 服务器端点

统一使用 `VDP_BASE_URL` 作为服务地址。

示例：

- 默认推荐（节点优选自动匹配）：`https://open-global.vdsentnet.com`

高级用户可以手动查看并选择更适合你的端点：

| base | 地区 | 服务提供商 |
| --- | --- | --- |
| `https://open-cn1.vdsentnet.com` | 中国上海 | Harry |

## 2. 默认认证方式（拿到签名后）

业务端点请求可使用以下任一方式：

1. `X-Api-Key: <apiKey>`
2. `Authorization: Bearer <accessToken>`

说明：

- 两者二选一。
- 同时传入时，服务端优先使用 `X-Api-Key`。

## 3. 接口特殊传参说明

部分接口还支持额外的传参方式（例如 Query、Body 或其他自定义 Header）。

- 如果接口文档有明确说明，以该接口文档为准。
- 如果接口文档没有说明，则本页的默认认证方式可直接使用。

## 4. 签名获取与换新

签名获取与换新流程请参考：

## 5. RequestID 返回规则

平台所有请求都会返回 `requestId`：

- Response Header：`x-request-id`
- JSON Body 字段：`requestId`
